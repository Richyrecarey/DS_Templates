CAP 1: TRATAMIENTO DE VARIABLES CATEGÓRICAS.¶

Los algoritmos entienden de números, con lo que para que el algoritmo pueda lidiar con las columnas categóricas, se tienen que transformar en numéricas.

sklearn posee directamente métodos para convertir estas variables en numéricas. Así, tenemos:

sklearn.preprocessing.LabelEncoder: Recibe un array de strings o enteros y nos devuelve uno de enteros con valores comrpendidos entre 0 y n-1, donde n es el número total de categorías de la variable.

El problema de labelencoder es que para más de dos clases, el algoritmo podrá entender que seguimos teniendo una relación de orden entre los datos, es decir: Una variable categórica que recoja puntos cardinales (N,S,E,O) indica diferentes valores pero el norte no es mayor ni menor que el oeste. Simplemente, es distinto. Si aplicamos labelencoder nos devolverá un array con valores (0,1,2,3). En algunos casos y algunos algoritmos pueden deducir que en estos datos existe una relación de orden, lo cual no es cierto.

Para evitar este problema, se recurre a sklearn.preprocessing.OneHotEncoder. Este genera n-1 variables "dummies" o binarias, es decir, que toman valores (0,1). Aquí ya se evita el que pueda inferirse un orden en las categorías, pues aquí sí que la variable significa 1== "Es norte", 0 ==" no es norte"" y así sucesivamente.

El problema que se presenta en este caso, es que si tenemos muchas variables categóricas con muchas categorías, el convertirlas en binarias nos aumenta notablemente el tamaño del dataset, con lo podemos incurrir en la "maldición de la dimensionalidad".

MÉTODOS DE REDUCCIÓN DE VARIABLES¶

Existen muchos métodos de reducción de variables existentes en sklearn.

  • Filtro por varianza: from sklearn.feature_selection import VarianceThreshold

Se define un umbral de varianza y todas las variables que no lo cumplan se eliminan. Muy útil para eliminar variables constantes. Dispone de los métodos fit y transform para aplicar a un dataset. El problema es que devuelve un array con las variables no eliminadas y no es sencillo rastrear cuáles ha eliminado. Normalmente hay que ir comparando columna a columna en el dataframe original y el array para ver cuáles ha eliminado y cuáles no.

  • Filtros univariantes basados en una clasificación de p-valores.

Según el modelo sea de clasificación o de regresión, se aplica un test estadístico (chi cuadrado, anova respectivametne) y tras indicar con cuántas variables queremos quedarnos, se crea una clasificación de variables y el modelo selecciona las k con los pvalores menores mayor grado de independencia entre variables)

from sklearn.feature_selection import SelectKBest from sklearn.feature_selection import f_regression

kb = SelectKBest(score_func = f_regression, # Pasamos únicamente el método, sin argumentos k=4 # Queremos quedarnos con las 4 variables mejores )

  • Selección basada en árbol de decisión.

Consiste en entrenar un árbol de decisión muy sobreajustado sobre todo el dataset y después quedarse con las variables que expliquen un valor determinado de la información: 90 %, 95 %....

Este método utiliza un modelo y. com overemos más adelante, todos los modelos de sklearn tienen los métodos:

  • .fit(X=conjunto de train de variables independientes, y=variable objetivo del conjunto de train)
  • .predict(X=conjunto de variables independientes). IMPORTANTE: tiene que tener las mismas variables que el que se utilizó para el .fit()

  • .score(y_real,y_predicción) Devuelve, para el caso de regresión, el r2 del modelo y para clasificación, la precisión entendida como % de aciertos sobre el total.

  • Selección basada en métodos recursivos.

Este caso funciona de modo similar a como lo hacen las regresiones "backward", es decir, se comienza probando todas las variables para ir sacando variables una a una. En sklearn existe la posibilidad de quedarse, como en el caso anterior, con las K-mejores variables (sklearn.feature_selection.RFE), o que el modelo automáticamente decida con cuántas se quiere quedar (sklearn.feature_selection.RFECV). En este último caso, sklearn utiliza la validación cruzada.

Estandarización y normalización¶

Los modelos basados en árboles son bastante estables en cuanto a las dimensiones de las variables. Sin embargo, otros modelos como las regresiones lineales, SVMS, redes necesitan que todas las variables estén dentro del mismo orden de magnitud. Para ello se puede utilizar standarscaler que reduce las variables a N(0,1) o maxminscaler que las incluye todas en el intervalo en que se indique, normalmente (0,1). No existe un criterio claro a la hora de seleccionar uno y otro. Como regla general, si las variables tienen muchos outliers es mejor recurrir a la estandarización y si están bastante centradas, a la normalización. Lógicamente, este procedimiento se aplica solo a las variables numéricas, no a las categóricas que previamente se han debido separar en binarias.

Cómo entrenar modelos.¶

A estas alturas, ya ha quedado mostrado con claridad que las clases de sklearn funcionan en general de la misma manera:

  • Método .fit para entrenar;
  • Método .transform para convertir;
  • Método .fit_transform que combina los anteriores.

En el caso de modelos, el uso es similar. Todos los modelos tienen los siguiente métodos:

  • Método .score para ver el resultado sobre un determinado conjunto del que se conoce el resultado final.

def curva_aprendizaje(modelo,X_train,X_test,y_train,y_test, paso=5000):
    from sklearn.metrics import mean_squared_error as mse
    train_errors=[]
    test_errors=[]
    #Vamos utilizando el train poco a poco, es decir, vamos a ir aumentándolo poco a poco y entrenando el modelo:
    for m in range(1,len(X_train),paso):
        modelo.fit(X_train[:m],y_train[:m])
        #Caluclamos el error en train:
        train_errors.append(mse(y_train[:m],modelo.predict(X_train[:m])))
        test_errors.append(mse(y_test,modelo.predict(X_test)))

    #Pintamos el gráfico.
    plt.plot(np.sqrt(train_errors),"r-",linewidth=2,label="Errores en train")
    plt.plot(np.sqrt(test_errors),"b-+",linewidth=3,label="Errores en test")

def rmse(modelo, X,y):
    from math import sqrt
    from sklearn.metrics import mean_squared_error
    return sqrt(mean_squared_error(modelo.predict(X),y))
def rmse_std(modelo, X,y,sc_target):
    from math import sqrt
    from sklearn.metrics import mean_squared_error
    return sqrt(mean_squared_error(sc_target.inverse_transform(modelo.predict(X)),sc_target.inverse_transform(y)))

VALIDACIÓN CRUZADA¶

La validación cruzada en dividir el conjunto de train en diferentes subgrupos (kfold o cvs en terminología sklearn) de modo que se entrena el modelo tantas veces como divisiones del conjunto se tengan. El modelo se entrena cada vez en n-1 divisiones y se utiliza la restante para validar. El resultado del modelo es el promedio resultante de validarlo en cada una de las iteraciones, dando lugar a un modelo más robusto.

La validación cruzada se combina con el "tuneo" de parámetros del modelo. Ambas cosas están incluidas en la clase GridSearch, que permite hacer ambas cosas simultáneamente. Después, con una serie de métodos, devuelve el mejor modelo, el resultado, etc..

Los parámetros asociados al modelo se deben pasar como diccionario

MODELOS DE CLASIFICACIÓN¶

El funcionamiento de modelos de clasificación es esencialmente el mismo que el de regresión. La única diferencia es que la variable objetivo debe ser una variable binaria o multiclase.

El tratamiento de ambas es esencialmente el mismo. La única exigencia es que la variable objetivo debe estar formada por enteros representativos de cada una de las clases.

TIPOS DE MODELOS DE CLASIFICACIÓN¶

Los modelos de clasificación se dividen en dos grupos:

  • Binarios: La variable objetivo es una variable con dos clases;
  • Multiclase: La variable categórica tiene más de dos.

Existen modelos como las redes neuronales, árboles, random forest... que pueden tratar problemas multiclase y binarios de igual forma.

Sin embargo, otros como las regresiones logísticas o SVM solamente permiten llevar a cabo clasificaciones binarias. Para poder adaptar estos modelos a multiclase, sklearn sigue dos tácticas diferentes:

  • "One vs all", también llamada "One vs rest" que entrena tantos clasificadores como clases tenga la variable objetivo y devuelve como resultado aquel que tenga un mejor resultado. Por ejemplo, si tenemos una variable con 3 clases, 0,1 y 2, entrenaría 3 modelos: 0 o no 0, 1 o no 1, 2 o no 2;

  • "one vs one" donde a diferencia del anterior, se entrenan clasificadores binarios para cada par de clases posibles. En el ejemplo anterior tendríamos 3 modelos: (0,1), (0,2) y (1,2). El problema de esta manera es que hay que entrenar un total de Nx(N-1)/2 clasificadores para clasificar N clases. La ventaja es que solo tienen que entrenarse en la porción del conjunto de train que se corresponde para esa clase.

En general, sklearn opta por la primera estrategia por defecto salvo en el caso de las SVMs que utiliza la segunda.

En cualquier caso, se puede elegir el método por el que se opta sin más que utilizar:

from sklearn.multiclass import OneVsRestClassifier from sklearn.multiclass import OneVsOneClassifier

Con las librerias importadas, se crea una isntancia indicando el modelo. Por ejemplo, si es un SVM lineal:

from sklearn.svm import LinearSVC

OneVsOneClassifier(LinearSVC(C=100.))

Después se utilizan los métodos .fit() etc... como si se tratara de un modelo normal.

MÉTRICAS DE CLASIFICACIÓN¶

En los proyectos de clasificación, existen diversas métricas disponibles además de la precisión del modelo sobre el conjunto de test.

Uno de los problemas habituales con los que uno se encuentra al trabajar en modelos de clasificación es el hecho de que las muestras suelen estar desequilibradas ("unbalanced") en el sentido en que una de las clases tiene una presencia mucho mayor que el resto.

Así, en casos de fraude es habitual tener una muestra mucho mayor de casos de no fraude, en mantenimiento predicitivo muchas menos situaciones de fallo que de no fallo etc.. Asimismo, no siempre lo que se busca no es tanto el acertar en un caso o en otro, sino que cuando se afirma que un caso es positivo, se proporcione con una gran precisión, más allá de que el precio sea dejarnos algún acierto por el camino.

Por estos motivos, no solo es conveniente analizar la precisión en sí, sino otras métricas relevantes como la sensibilidad, , etc.. y contrastar resultados mediante la matriz de confusión y curva ROC etc...

sklearn permite obtener de forma sencilla todas estas métricas gracias a classification_report
