CAP 1: TRATAMIENTO DE VARIABLES CATEGÓRICAS.¶

Los algoritmos entienden de nmeros, con lo que para que el algoritmo pueda lidiar con las columnas categricas, se tienen que transformar en numricas.

sklearn posee directamente mtodos para convertir estas variables en numricas. As, tenemos:

sklearn.preprocessing.LabelEncoder: Recibe un array de strings o enteros y nos devuelve uno de enteros con valores comrpendidos entre 0 y n-1, donde n es el nmero total de categoras de la variable.

El problema de labelencoder es que para ms de dos clases, el algoritmo podr entender que seguimos teniendo una relacin de orden entre los datos, es decir: Una variable categrica que recoja puntos cardinales (N,S,E,O) indica diferentes valores pero el norte no es mayor ni menor que el oeste. Simplemente, es distinto. Si aplicamos labelencoder nos devolver un array con valores (0,1,2,3). En algunos casos y algunos algoritmos pueden deducir que en estos datos existe una relacin de orden, lo cual no es cierto.

Para evitar este problema, se recurre a sklearn.preprocessing.OneHotEncoder. Este genera n-1 variables "dummies" o binarias, es decir, que toman valores (0,1). Aqu ya se evita el que pueda inferirse un orden en las categoras, pues aqu s que la variable significa 1== "Es norte", 0 ==" no es norte"" y as sucesivamente.

El problema que se presenta en este caso, es que si tenemos muchas variables categricas con muchas categoras, el convertirlas en binarias nos aumenta notablemente el tamao del dataset, con lo podemos incurrir en la "maldicin de la dimensionalidad".

MÉTODOS DE REDUCCIÓN DE VARIABLES¶

Existen muchos mtodos de reduccin de variables existentes en sklearn.

  • Filtro por varianza: from sklearn.feature_selection import VarianceThreshold

Se define un umbral de varianza y todas las variables que no lo cumplan se eliminan. Muy til para eliminar variables constantes. Dispone de los mtodos fit y transform para aplicar a un dataset. El problema es que devuelve un array con las variables no eliminadas y no es sencillo rastrear cules ha eliminado. Normalmente hay que ir comparando columna a columna en el dataframe original y el array para ver cules ha eliminado y cules no.

  • Filtros univariantes basados en una clasificacin de p-valores.

Segn el modelo sea de clasificacin o de regresin, se aplica un test estadstico (chi cuadrado, anova respectivametne) y tras indicar con cuntas variables queremos quedarnos, se crea una clasificacin de variables y el modelo selecciona las k con los pvalores menores mayor grado de independencia entre variables)

from sklearn.feature_selection import SelectKBest from sklearn.feature_selection import f_regression

kb = SelectKBest(score_func = f_regression, # Pasamos nicamente el mtodo, sin argumentos k=4 # Queremos quedarnos con las 4 variables mejores )

  • Seleccin basada en rbol de decisin.

Consiste en entrenar un rbol de decisin muy sobreajustado sobre todo el dataset y despus quedarse con las variables que expliquen un valor determinado de la informacin: 90 %, 95 %....

Este mtodo utiliza un modelo y. com overemos ms adelante, todos los modelos de sklearn tienen los mtodos:

  • .fit(X=conjunto de train de variables independientes, y=variable objetivo del conjunto de train)
  • .predict(X=conjunto de variables independientes). IMPORTANTE: tiene que tener las mismas variables que el que se utiliz para el .fit()

  • .score(y_real,y_prediccin) Devuelve, para el caso de regresin, el r2 del modelo y para clasificacin, la precisin entendida como % de aciertos sobre el total.

  • Seleccin basada en mtodos recursivos.

Este caso funciona de modo similar a como lo hacen las regresiones "backward", es decir, se comienza probando todas las variables para ir sacando variables una a una. En sklearn existe la posibilidad de quedarse, como en el caso anterior, con las K-mejores variables (sklearn.feature_selection.RFE), o que el modelo automticamente decida con cuntas se quiere quedar (sklearn.feature_selection.RFECV). En este ltimo caso, sklearn utiliza la validacin cruzada.

Estandarización y normalización¶

Los modelos basados en rboles son bastante estables en cuanto a las dimensiones de las variables. Sin embargo, otros modelos como las regresiones lineales, SVMS, redes necesitan que todas las variables estn dentro del mismo orden de magnitud. Para ello se puede utilizar standarscaler que reduce las variables a N(0,1) o maxminscaler que las incluye todas en el intervalo en que se indique, normalmente (0,1). No existe un criterio claro a la hora de seleccionar uno y otro. Como regla general, si las variables tienen muchos outliers es mejor recurrir a la estandarizacin y si estn bastante centradas, a la normalizacin. Lgicamente, este procedimiento se aplica solo a las variables numricas, no a las categricas que previamente se han debido separar en binarias.

def curva_aprendizaje(modelo,X_train,X_test,y_train,y_test, paso=5000):
    from sklearn.metrics import mean_squared_error as mse
    train_errors=[]
    test_errors=[]
    #Vamos utilizando el train poco a poco, es decir, vamos a ir aumentndolo poco a poco y entrenando el modelo:
    for m in range(1,len(X_train),paso):
        modelo.fit(X_train[:m],y_train[:m])
        #Caluclamos el error en train:
        train_errors.append(mse(y_train[:m],modelo.predict(X_train[:m])))
        test_errors.append(mse(y_test,modelo.predict(X_test)))

    #Pintamos el grfico.
    plt.plot(np.sqrt(train_errors),"r-",linewidth=2,label="Errores en train")
    plt.plot(np.sqrt(test_errors),"b-+",linewidth=3,label="Errores en test")

def rmse_std(modelo, X,y,sc_target):
    from math import sqrt
    from sklearn.metrics import mean_squared_error
    return sqrt(mean_squared_error(sc_target.inverse_transform(modelo.predict(X)),sc_target.inverse_transform(y)))

VALIDACIÓN CRUZADA¶

La validacin cruzada en dividir el conjunto de train en diferentes subgrupos (kfold o cvs en terminologa sklearn) de modo que se entrena el modelo tantas veces como divisiones del conjunto se tengan. El modelo se entrena cada vez en n-1 divisiones y se utiliza la restante para validar. El resultado del modelo es el promedio resultante de validarlo en cada una de las iteraciones, dando lugar a un modelo ms robusto.

La validacin cruzada se combina con el "tuneo" de parmetros del modelo. Ambas cosas estn incluidas en la clase GridSearch, que permite hacer ambas cosas simultneamente. Despus, con una serie de mtodos, devuelve el mejor modelo, el resultado, etc..

Los parmetros asociados al modelo se deben pasar como diccionario

MODELOS DE CLASIFICACIÓN¶

El funcionamiento de modelos de clasificacin es esencialmente el mismo que el de regresin. La nica diferencia es que la variable objetivo debe ser una variable binaria o multiclase.

El tratamiento de ambas es esencialmente el mismo. La nica exigencia es que la variable objetivo debe estar formada por enteros representativos de cada una de las clases.

TIPOS DE MODELOS DE CLASIFICACIÓN¶

Los modelos de clasificacin se dividen en dos grupos:

  • Binarios: La variable objetivo es una variable con dos clases;
  • Multiclase: La variable categrica tiene ms de dos.

Existen modelos como las redes neuronales, rboles, random forest... que pueden tratar problemas multiclase y binarios de igual forma.

Sin embargo, otros como las regresiones logsticas o SVM solamente permiten llevar a cabo clasificaciones binarias. Para poder adaptar estos modelos a multiclase, sklearn sigue dos tcticas diferentes:

  • "One vs all", tambin llamada "One vs rest" que entrena tantos clasificadores como clases tenga la variable objetivo y devuelve como resultado aquel que tenga un mejor resultado. Por ejemplo, si tenemos una variable con 3 clases, 0,1 y 2, entrenara 3 modelos: 0 o no 0, 1 o no 1, 2 o no 2;

  • "one vs one" donde a diferencia del anterior, se entrenan clasificadores binarios para cada par de clases posibles. En el ejemplo anterior tendramos 3 modelos: (0,1), (0,2) y (1,2). El problema de esta manera es que hay que entrenar un total de Nx(N-1)/2 clasificadores para clasificar N clases. La ventaja es que solo tienen que entrenarse en la porcin del conjunto de train que se corresponde para esa clase.

En general, sklearn opta por la primera estrategia por defecto salvo en el caso de las SVMs que utiliza la segunda.

En cualquier caso, se puede elegir el mtodo por el que se opta sin ms que utilizar:

from sklearn.multiclass import OneVsRestClassifier from sklearn.multiclass import OneVsOneClassifier

Con las librerias importadas, se crea una isntancia indicando el modelo. Por ejemplo, si es un SVM lineal:

from sklearn.svm import LinearSVC

OneVsOneClassifier(LinearSVC(C=100.))

Despus se utilizan los mtodos .fit() etc... como si se tratara de un modelo normal.

As, en casos de fraude es habitual tener una muestra mucho mayor de casos de no fraude, en mantenimiento predicitivo muchas menos situaciones de fallo que de no fallo etc..
